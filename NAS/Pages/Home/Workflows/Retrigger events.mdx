Learn how to rerun past events so you can retrigger the actions of your workflows – useful if you want to test a workflow that previously failed.

Reflow by multiple IDs
----------------------

Use the following endpoint to retrigger past events associated with multiple event IDs and workflow IDs, or multiple subject IDs (for example, payment IDs and dispute IDs) and workflow IDs.

You can find the [full specification in our API reference](https://api-reference.checkout.com/preview/crusoe/#tag/Workflows/paths/~1workflows~1events~1reflow/post).

Reflow by event ID
------------------

Pass an event's ID in the path of the following endpoint to rerun a single past event. This will trigger the actions of any existing workflows that are conditional on that event.

You can find the [full specification in our API reference](https://api-reference.checkout.com/preview/crusoe/#tag/Workflows/paths/~1workflows~1events~1{eventId}~1reflow/post).

### The request

#### Endpoints

*   [**Live**](#tabs-c524925d-e684-4d13-beb0-a8623c3c3201-1)
*   [**Sandbox**](#tabs-c524925d-e684-4d13-beb0-a8623c3c3201-2)

POST`https://api.checkout.com/workflows/events/{eventId}/reflow`

POST`https://api.sandbox.checkout.com/workflows/events/{eventId}/reflow`

Reflow by event and workflow ID
-------------------------------

You can find the [full specification in our API reference](https://api-reference.checkout.com/preview/crusoe/#tag/Workflows/paths/~1workflows~1events~1{eventId}~1workflow~1{workflowId}~1reflow/post).

Reflow by subject ID
--------------------
