These APIs can be used to create, retrieve, update, and delete information in a customer's card object.

The customer object is an outline of all the information stored about a customer. It consists of the following fields:

x

|     |     |
| --- | --- |

|
\`id\` <span class="field-type">String</span>

|

A string of characters (prefixed with cust\_) that uniquely identifies the customer.

|
|

\`card\` <span class="field-type">Hash</span>

|

A hash containing an array of \[cards\](doc:cards).

|
|

\`count \` <span class="field-type">Integer</span>

|

The number of card objects included in the array.

|
|

\`created\` <span class="field-type">String</span>

|

The UTC date and time expressed according to ISO 8601. (e.g., \*\*2015-11-05T13:10:30Z\*\* or \*\*2015-11-05T08:10:30-05:00\*\*)

|
|

\`data\` <span class="field-type">Array</span>

|

An array of card objects.

|
|

\`defaultCard\` <span class="field-type">String</span>

|

The card ID of the customer's default card.

|
|

\`description\` <span class="field-type">String</span>

|

The description specified in the request.

|
|

\`email\` <span class="field-type">String</span>

|

The customer's email address.

|
|

\`liveMode\` <span class="field-type">Boolean</span>

|

Defined as: \* \`true\` if the live keys were used in the request. \* \`false\` if the test keys were used in the request.

|
|

\`name \` <span class="field-type">String</span>

|

The customer's name.

|
|

\`metadata\` <span class="field-type">Hash</span>

|

A hash of FieldName and value pairs (e.g., {"keys1": "Value1"}).

|
|

\`phone\` <span class="field-type">Object</span>

|

Phone number of the customer containing: \* \`countryCode\`: Valid country code for the phone number (e.g. 44 for the United Kingdom) \* \`number\`: Phone number.

|
