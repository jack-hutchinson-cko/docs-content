After authorizing a card payment, the actual transaction is not complete until it has been captured. Capture time depends on the merchant's settings,—but assuming it is not instant—then it is possible to void a transaction either in \[The Hub\](https://hub.checkout.com) or using the following API request. This endpoint voids an authorized, but not yet captured, card payment. 

Our Void API request can be used on your server-side code to set up a payment request. New to our APIs? Read our \[API quickstart\](doc:api-quickstart) first! ## The request Requests are created using a POST request. Use the details below to set up your request. ### Endpoint live <pre class="api-header-long-url"><p class="api-header"><span class="post-tag">POST</span>https://api2.checkout.com/v2/charges/{chargeId}/void</p></pre> ### Endpoint sandbox <pre class="api-header-long-url"><p class="api-header"><span class="post-tag">POST</span>https://sandbox.checkout.com/api2/v2/charges/{chargeId}/void</p></pre> ### Header and path parameters

\## The response You’re all done! Congrats! A successful void request returns a void payment response — we've provided an example of one below. Be sure to contact our \[support team\](mailto:support@checkout.com) if you run into any problems. 

The \`id\` is the unique identifier of the transaction and the \`originalId\` is the id of the parent transaction (for example, in a captured transaction, the \`originalId\` will be the id of the original authorised transaction).
