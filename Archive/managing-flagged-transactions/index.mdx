Checkout.com highly recommends researching each flagged transaction carefully before taking any action.

If the Risk Settings configured for your business trigger a flagged transaction, you must either capture or void it manually. If you take no action, the transaction will expire automatically after 7 days. Flagged transactions can be identified in The Hub's Transactions Screen by an orange **Authorisation:Flag** status and a **10100** response code. This response code denotes that the transaction has been authorized, but it will not progress to the capture stage automatically even if auto capture has been enabled.

If you determine that the flagged transaction is valid, tick the checkbox next to the transaction and choose Capture from the Action drop-down menu. Select _Full Capture_ or _Partial Capture_, enter a note (optional) and click _Capture_ to complete the transaction.

---

![](https://files.readme.io/ca6254b-managing-flagged-transactions-capture-transaction.png)

![](https://files.readme.io/ca6254b-managing-flagged-transactions-capture-transaction.png)**managing-flagged-transactions-capture-transaction.png**

If you determine that the flagged transaction is invalid or in any way illegitimate, tick the checkbox next to the transaction and choose _Void_ from the Action drop-down menu. Enter a note (optional) and click _Void_ to complete the transaction.
