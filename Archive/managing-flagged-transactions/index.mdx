Checkout.com highly recommends researching each flagged transaction carefully before taking any action.

If the Risk Settings configured for your business trigger a flagged transaction, you must either capture or void it manually. If you take no action, the transaction will expire automatically after 7 days. Flagged transactions can be identified in The Hub's Transactions Screen by an orange \*\*Authorisation:Flag\*\* status and a \*\*10100\*\* response code. This response code denotes that the transaction has been authorized, but it will not progress to the capture stage automatically even if auto capture has been enabled.
