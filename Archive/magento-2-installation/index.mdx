\*With our Magento 2 installation, you can quickly start accepting payments on the Checkout.com payment gateway.\* 

1\. Navigate to your Magento 2 root directory and run the following command: \`\`\` composer require checkoutcom/magento2:\* \`\`\` 2. Recompile your Magento 2 installation and clear the cache: \`\`\` bin/magento setup:upgrade rm -rf var/cache var/generation/ var/di bin/magento setup:di:compile && php bin/magento cache:clean \`\`\` \*For additional information on the installation process, please refer to Magento 2’s full developer \[guide\](http://devdocs.magento.com/guides/v2.0/install-gde/install/cli/install-cli-subcommands-enable.html).\* 

1\. \*\*Find the configuration form\*\*. Once the extension has been successfully installed, navigate to your Magento 2 dashboard; from here, we can locate the payment configuration form. a. Click \*\*Stores\*\* in the navigation menu (on the left-hand side of the screen) and select \*\*Configuration\*\* from the drop-down menu. b. In the \*\*Configuration\*\* menu, find \*\*Sales\*\* and from the subcategories click \*\*Payment Methods\*\*. You should now see the Payment Methods configuration page. 2. \*\*Open the configuration form\*\*. a. Click \*\*OTHER PAYMENT METHODS\*\* and select \*\*Checkout.com\*\*.

The \*\*Private Shared Key\*\* is a header that The Hub sends to your page with every transaction. It is not required, but where possible should be specified. The Private Shared Key is used to check the authenticity of notification WebHooks coming from Checkout.com. 5. \*\*Save your settings\*\* Once the account keys have been added, ensure that you save your changes using the \*\*Save Config\*\* button. \*\*That's it!\*\* You're all set to start taking payments.
