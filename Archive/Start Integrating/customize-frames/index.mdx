One of our favorite things about Frames is its customizability. By following this styling guide, you can make changes to just about everything.

To start, let's look at the payment form itself. Frames comes in two variants: standard and simple. ## Standard view The standard view is the default template for Frames. It is a shorter two-row layout with icons depicting each field.

---

![](https://files.readme.io/20f5546-standard-theme-dimensions.png)

![](https://files.readme.io/20f5546-standard-theme-dimensions.png)**standard-theme-dimensions.png**

Although not necessary (as it is default), to specify the **standard theme** use the `theme` attribute in your code. For example, `theme: 'standard',`. ## Simple view Use the theme attribute to change to the _simple_ version of Frames. Simple view is a taller three-row layout with icons depicting each field.

---

![](https://files.readme.io/80427d9-simple-theme-dimensions.png)

![](https://files.readme.io/80427d9-simple-theme-dimensions.png)**simple-theme-dimensions.png**

To use the **simple view**, define the theme attribute in your code as `theme: 'simple',`.

Frames can be customized to fit seamlessly with your brand. By using a `style` variable within the Frames script itself, it's easy to start making changes. All customizations are achieved using **selectors**, **properties**, and **values** in a manner very familiar to developers. Use the tabs below to view an example of styling, and a reference of selectors to use when styling your version of Frames.

Due to stringent PCI compliance rules, Frames can only support Web-safe fonts.

Make your customers feel at home by using our localization settings to change the language of your payment form. Use one of our pre-defined languages or create your own. ### Pre-defined localization If the language you would like to use appears in our pre-defined list, then just set the `localisation` parameter, for example `localisation: 'FR-FR'`. The pre-defined languages: English = **EN-GB** Dutch = **NL-NL** French = **FR-FR** German = **DE-DE** Italian = **IT-IT** Korean = **KR-KR** Spanish = **ES-ES** ### Custom localization You can customize the language of both the placeholders of the form's input fields and the card number error message. If you would like to use a language that is not a part of any of the pre-defined localization language sets, you can define your own localization fields. To define the language of your form, use the `localisation` variable with the following properties and custom values. The example below shows the localisation variable in action:

JavaScript

// custom localisation
var localisation = {

cardNumberPlaceholder: "Custom text",

expiryMonthPlaceholder: "Custom text",

expiryYearPlaceholder: "Custom text",

cvvPlaceholder: "Custom text",
cardNumberErrorMessage: "Custom text"

8

}

9

The example above can be used in your own Frames localization, simply replace the text "Custom text" with your own translation or wording.
