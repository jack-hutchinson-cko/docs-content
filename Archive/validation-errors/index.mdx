If an error occurs while validating a request, the API responds with a `400 HTTP` response code, accompanied by a JSON response containing the error details. Find out more in the example below. The JSON response includes the following fields:

Add Row Add Col

|     |     |
| --- | --- |

|
`errorCode`

|

The code associated with the error.

|
|

`message`

|

The message describing the error.

|
|

`errors`

|

An array of validation errors.

|
|

`errorMessageCodes`

|

The specific codes associated with the errors.

|
|

`eventId`

|

The error event ID used by Checkout.com to trace what went wrong in the transaction.

|

​ #### Request example Using a JSON payload with the `value` and `currency` fields missing.

cURL

curl https://sandbox.checkout.com/api2/v2/charges/token

-H "Authorization: sk_test_55aedccc-7f53-4ccc-b0a6-d943decc3c31"

-H "Content-Type:application/json;charset=UTF-8"

-X POST

-d '{

"cardtoken" : "card_tok_cf16c4d0-af33-4a10-80fa-806839ae0179"

}'

#### Response example

JSON
{

"errorCode": "70000",

"message": "Validation error",

"errors": [

"Require either a 'customerId' or 'email' value",
"Field 'value' is required",
"Field 'currency' is required"
],
"errorMessageCodes": [
"70055",
"70049",
"70051"
],
"eventId": "1beadef0-458f-40ff-84fe-00e3d38e228c"
}

If an error occurs while validating certain parts of the transaction details, the API responds with a `400 HTTP` response code, accompanied by a JSON response containing the error details. The JSON response contains the following fields:​

|     |     |

`errorCode`

|

|

|

The message describing the error

|
|

`eventId`

|

The error event ID for Checkout.com to trace what went wrong in the transaction.

|

#### Request example The example below shows a scenario where the request payload is valid, but the IP address is invalid on the gateway.

cURL

curl https://sandbox.checkout.com/api2/v2/charges

-H "Authorization: sk_test_55aedccc-7f53-4ccc-b0a6-d943decc3c31"

-H "Content-Type:application/json;charset=UTF-8"

-H "Accept:application/json"

-X POST

-d'{
"cardtoken" : "card_tok_C123C0A5-F850-453A-92DE-93607DD6A916",
"currency" : "USD",
"email" : "LanaSMartin@dayrep.com",
"value" : 100,
"customerIp": "abc"
}'

#### Response example

JSON

{
"errorCode": "82051",
"message": "Invalid Customer IP",
"eventId": "cf0acbed-5b46-4a8b-85db-9372b30dfca1"
}
